Week 2 Assignment

Part 1: Create a GitHub Pages Landing Page for Assignments This Semester

Before starting this assignment, you will create a basic landing page on GitHub Pages that will serve as a homepage for all future assignments.

  1. For this, you will be updating the code in your index.html to include a few basic HTML elements to allow for navigation to your various projects throughout the semester.
  2. Webpage Title: Set the website page title as “WEB 110 Projects” 
  3. Wireframe Header Area: Code the header element with the text “WEB 110 Projects” within an <h1> heading element.
  4. Wireframe Navigation Area: Place the following text within a <nav> element with bold text (<b> element):
    Home | Parks & Rec
    1. Add extra blank spaces between the hyperlinks with the &nbsp; special character as needed
    2. Link the Home link to "index.html" and the Parks & Rec link to "Week2Assignment/home.html". This will be your home page for the Week 2 Assignment.

Here is an example of what this will look like: https://github.com/alexasummers/WEB110Fall2025/blob/main/index.html Links to an external site.

We will continue adding to this page as the semester goes on. Each time you add a new assignment to the landing page, you will link in the nav bar to the homepage for that assignment.

 

Part 2: Parks & Rec Website: Home Page

Website Information: This website will provide information about your local parks and recreation programs. The target audience is residents who want to explore available parks, recreational activities, and community events. The site will include details about park locations and details.

Procedure: You will create two web pages for your Local Parks and Recreation website: a Home page and a Parks page.

 

  1. Create a folder for our class on your computer called "WEB110" with a folder called "Week2" inside of it (see picture). Be sure to remember the location of this folder.
  2. Open your text editor, set the language of your file to HTML, and save this to the Week2 folder as "home.html". This will be your homepage.

 

 

 

We will now start creating the home page. It will turn out to be something like this: 

1) Webpage Title: Set the website page title as “Local Parks and Recreation.” (hint: this is the tab of the page: )

2) Wireframe Header Area: Code the header element with the text “Welcome to Local Parks and Recreation” within an <h1> heading element.

3) Wireframe Navigation Area: Place the following text within a <nav> element with bold text (<b> element):
Home | Parks

Then, Code anchor tags so that “Home” links to home.html, and “Parks” links to parks.html.  Add extra blank spaces between the hyperlinks with the &nbsp; special character as needed.

4) Wireframe Main Content Area: Code the page content within a <main> element, including:

a. Place the following within an h2 element: “Discover Your Local Parks and Recreation Programs.”
b. Place the following content in a paragraph: “Our community is home to a variety of parks and programs designed for all ages. From scenic walking trails to fun activities, there’s something for everyone!”
c. Place the following content into a second paragraph: “Learn about recreational activities, local events, and ways to get involved. Whether you’re looking for outdoor fun or community programs, we have it all! Here is a little of what we have to offer:"
d. Place the following in an unordered list: 

  • Archery
  • Water activities
  • Birding
  • Shelter rentals
  • Fishing

5) Wireframe Footer Area

Use a <footer> element to include the copyright and email address. Use a small text size (<small> element) and italic font style (<em> element). The copyright information is:

Copyright © 2023 Local Parks and Recreation

Use a line break (<br>), and then on the line below, write the word "Contact: " and put your email address with an email hyperlink.

Lastly, add an <a> tag with the text "Return to landing page", and link back to your main index.html page (hint: use ../index.html to return back to the folder you originally came from, and then select index.html)

Make sure to save your file!!

Part 3: Local Parks & Rec Page

We will now start working on the parks page. It will turn out to be something like this: 

 

1) A productivity technique is to create new pages based on existing pages so you can benefit from your previous work. Your new Parks page will use the home.html page as a starting point. Open the home.html page in a text editor. Select File > Save As and save the file with the new name parks.html in your Week 2 folder.

Now you are ready to edit the page!


2) Web Page Title: Modify the page title. Change the text between <title> and </title> tags to Parks. The tab will look like this:

3) Wireframe Main Content Area:

a. Replace the text within the <h2> tags with: Explore our Parks.

b. Delete: the paragraphs, unordered list, and the contact information from the homepage.

c. Create: a description list for parks using a description list. Remember, a description list needs to be inside of a <dl> element. Use the <dt> element to contain each park name. Configure the park name to display in bold text (use the <strong> element). Use the <dd> element to contain the description of the park. The parks and descriptions are as follows:

Central Park
A peaceful retreat with walking trails, picnic areas, and a playground.

Sunset Lake Park
Enjoy fishing, kayaking, and scenic sunsets by the lake.

Evergreen Sports Complex
Home to basketball courts, soccer fields, and an indoor gym.

4) Wireframe Footer Area:

Keep your footer area exactly the same.

Save your file!! Make sure it is saving as parks.html and not home.html, or it will overwrite your home page.

Preview it on your computer: Open the file on your computer in a browser and confirm it looks like the snapshots above.  Test the hyperlink from the parks.html page to the home.html. Test the hyperlink from the home.html page to parks.html. If your links do not work, review your work with close attention to these details:

  • Verify that you have saved the pages with the correct names in the correct folder.
  • Verify your spelling of the page names in the anchor tags.
  • After you make changes, test again.

Once you know that your pages are linking to each other, check your work in the W3C HTML validator. https://validator.w3.org/ Links to an external site.

Be sure to check for spelling as well. The easiest way to do this is to open your files in the browser and to copy all the text into a word editor.

Uploading your work: 

Upload your work to a new folder in GitHub called "Week2Project". This folder should be created in your WEB110 Repository (the original repository you created for this class).

Once your files are done uploading and the workflow is done running, you should be able to view your website by going to your main index.html page on the GitHub pages website. The link you need to go to should be something like:

https://YourUserName.github.io/YourRepoName/index.html

 

This is the link I need you to submit to Canvas. Make sure to send me the link to your working website, and NOT to your repository.

Submitting your work: Copy your website link from the browser window. Go to Canvas, Start the assignment for Week 2 and paste that link in the Week 2 project box at the bottom of the page. Then, submit your assignment.
